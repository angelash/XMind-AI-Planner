# 自动化运行说明（连续执行直到完成）

## 1. 目标

对 `automation_tasks.yaml` 中的任务进行持续调度，直至全部任务为 `done`，且 `REL-01` 完成。

## 2. 调度规则

1. 每次检查任务状态。
2. 若存在 `developing` 任务：不启动新任务，继续等待。
3. 若不存在 `developing` 任务：选择第一个“依赖已满足且状态为 `waiting`”的任务启动为 `developing`。
4. 任务执行完成后按流程推进：`diff_ready -> sync_ok -> build_ok -> done`。
5. 任务失败标记为 `failed`，可重置为 `waiting` 重跑；连续失败时标记 `need_confirm`。
6. 当所有任务为 `done` 且 `REL-01` 通过，停止调度。

## 3. 轮询间隔

- 轮询周期：5 分钟
- 不区分白天/夜晚，不做人工排班

## 4. 人工确认触发点

1. 任务连续失败。
2. 发生高风险文件删除或大规模覆盖。
3. 依赖冲突导致无法自动推进。
