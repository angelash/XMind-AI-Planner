## 1. 项目背景



策划人员在编写策划案时，通常需要从一个核心概念出发，逐步细化展开。传统方式是手动编写文档，效率低且容易遗漏。本项目旨在提供一个**原生思维导图编辑器**，结合 AI 能力，帮助策划人员快速构建和细化策划案。



## 2. 核心理念



**不是导入 XMind，而是操作形态像 XMind**



- 用户可以从零创建思维导图

- 每个节点可以手动输入或让 AI 生成内容

- AI 可以基于当前节点展开生成下级节点

- 逐步细分细化，构建完整的策划案

#### 3.4.1 独立导出功能



支持将指定节点标记为"独立导出"，导出 Markdown 时自动拆分为多文件：



| 功能 | 描述 | 优先级 |

|------|------|--------|

| 标记独立导出 | 右键菜单或工具栏按钮标记节点 | P0 |

| 取消标记 | 再次点击取消独立导出标记 | P0 |

| 视觉标记 | 已标记节点显示 📎 图标，右键菜单显示 ✅ 表示已标记 | P0 |

| 多文件导出 | 导出时生成 ZIP 包，包含主文件和独立文件 | P0 |

| 链接引用 | 主文件中用 Markdown 链接引用独立文件 | P0 |



#### 3.4.2 关联图片导出功能



导出 Markdown 时自动识别并打包备注中引用的本地图片，确保导出的 MD 在离线环境下图片可正常显示：



| 功能 | 描述 | 优先级 |

|------|------|--------|

| 图片引用识别 | 递归扫描所有节点备注，提取 `/uploads/images/...` 格式的图片引用 | P0 |

| 图片路径重写 | 将 MD 中的服务器路径 `/uploads/images/2026/02/hash.ext` 重写为相对路径 `./images/hash.ext` | P0 |

| 图片文件打包 | 将引用的图片文件复制到 ZIP 包的 `images/` 目录 | P0 |

| 自动 ZIP 升级 | 无图片且无独立导出时返回单 .md 文件；有图片或独立导出时自动升级为 .zip 包 | P0 |

| 缺失图片容错 | 图片文件不存在时跳过并记录日志，不阻断导出 | P0 |



**ZIP 包结构（含图片）：**

```

导图标题.zip

├── 导图标题.md          # 主 MD（图片路径已重写为 ./images/xxx）

├── 独立节点A.md         # 可选，独立导出节点（图片路径同样重写）

├── 独立节点B.md         # 可选

└── images/              # 所有关联图片

    ├── abc123def.png

    ├── 789xyz.jpg

    └── ...

```



**图片引用路径对照：**



| 环节 | 路径格式 |

|------|----------|

| 上传存储（磁盘） | `uploads/images/{year}/{month}/{md5_hash}.{ext}` |

| 备注中引用（在线） | `![alt](/uploads/images/2026/02/hash.ext)` |

| 导出 ZIP 内 MD | `![alt](./images/hash.ext)` |

| 导出 ZIP 内图片 | `images/hash.ext` |

## 16. 自动化开发闭环（v3.2）



### 16.1 功能概述



在思维导图中编写需求描述，AI 自动生成代码变更，形成「策划 → 开发 → 测试」的闭环。



### 16.2 核心流程



```

脑图节点变更 → 自动开发任务入队 → AI 分析需求 → AI 生成代码 → 

Diff 预览 → 用户确认 → 同步到测试空间 → 构建验证 → 完成

```



### 16.3 任务队列系统



| 功能 | 描述 | 优先级 |

|------|------|--------|

| 任务入队 | 脑图节点变更时自动创建开发任务 | P0 |

| 状态机 | waiting→coding→diff_ready→sync_ok→build_ok→done | P0 |

| 串行执行 | 每个工作区同一时间只执行一个任务 | P0 |

| need_confirm | 删除/大量覆盖/需求不清时触发人工确认 | P0 |

| 前置校验 | 启用自动开发前校验：文档必须在工作区中、工作区必须配置代码根目录和导出根目录 | P0 |

| 任务操作 | 取消（coding 中）、重跑、回滚（done 后） | P0 |



### 16.4 状态机定义



| 状态 | 说明 | 后续状态 |

|------|------|----------|

| waiting | 等待执行 | coding, canceled |

| coding | AI 生成代码中 | diff_ready, need_confirm, failed |

| need_confirm | 需要人工确认 | coding, canceled |

| diff_ready | Diff 已生成，待同步 | sync_ok, failed |

| sync_ok | 已同步到测试空间 | build_ok, failed |

| build_ok | 构建验证通过 | done |

| done | 完成 | rolled_back |

| failed | 失败 | waiting (重跑) |

| canceled | 已取消 | - |

| rolled_back | 已回滚 | - |
