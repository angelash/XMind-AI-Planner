# 项目设计文档 - XMind AI Planner（第 1 部分）



## 1. 系统架构



系统分为三层。**前端**在浏览器中运行，包含页面结构、主题样式、业务逻辑与思维导图库，业务层与思维导图库之间通过数据格式转换衔接；前端与后端通过 HTTP 或 REST 通信。**后端**采用 FastAPI，提供静态文件服务、各类 API 路由以及 AI 服务层（兼容 OpenAI 接口）；后端与外部 AI 服务通过 HTTPS 通信。**最底层**为 AI 服务（如 OpenAI 或其他兼容接口）。



## 2. 技术栈



- **前端 UI**：HTML5 与 CSS3，原生实现，无前端框架依赖。

- **前端交互**：原生 JavaScript（ES 模块），负责业务逻辑与 MindElixir 适配层。

- **思维导图**：MindElixir，本地存放在静态资源目录下，负责画布、布局、拖拽、缩放、连线、折叠等，无需外网。

- **后端框架**：Python FastAPI，异步高性能。

- **AI 接口**：OpenAI 兼容 API，支持多种 AI 服务。

- **文档导出**：python-docx，用于生成 Word 文档。



## 3. 数据结构



### 3.1 思维导图节点 (Node)



思维导图节点包含以下字段：**id** 为唯一标识，格式为“node-”加若干字符；**text** 为节点内容；**memo** 为可选备注，有备注的节点在界面上显示💬图标；**exportSeparate** 为可选的独立导出标记，已标记的节点显示📎图标；**children** 为可选的子节点数组。



### 3.2 前端状态



前端维护的全局状态包括：MindElixir 实例（负责渲染与交互）、当前选中节点 ID（与 MindElixir 的选中态同步）。布局、缩放、平移、折叠等由 MindElixir 内部管理，前端无需单独维护布局结构。



### 3.3 与 MindElixir 的数据格式转换



后端与本地存储使用“我们的格式”，MindElixir 使用“库格式”，前端负责双向转换：节点在我们这边用 id、text、children 表示，在库中用 id、topic、根标记、children 表示；整图初始化时由我们的根节点转换为库所需的节点数据与方向；导出或保存时从库的取数接口得到数据再转回我们的格式。



## 4. API 接口设计



### 4.1 配置管理



提供获取当前 AI 配置的接口（返回时 API Key 脱敏），以及更新 AI 配置的接口；扩展字段包括目录树全局剔除规则，支持通配符。



### 4.2 AI 功能



- 生成初始思维导图结构：请求需携带主题参数，返回是否成功及脑图节点。

- 展开生成子节点：请求需携带节点 ID、节点文本、路径、已有子节点及可选上下文，返回是否成功及子节点列表。

- 重新生成节点内容：请求需携带节点 ID、节点文本、路径及兄弟节点等信息，返回是否成功及新文本。

- 生成节点备注：请求体包含节点 ID、节点文本、路径及可选上下文，返回是否成功及备注内容。



### 4.3 导出功能



- 导出 Markdown：请求体为脑图节点。若无独立导出节点且无图片引用，则返回单个 MD 文件，文件名为导图标题；若有独立导出节点或有图片引用，则返回 ZIP 包，内含主 MD、独立节点 MD 及 images 目录。

- 导出 Word：请求体为脑图节点，返回 Word 文件下载。



#### 独立导出机制



节点数据中增加“是否独立导出”的布尔字段。右键菜单或工具栏可切换该标记；已标记节点显示📎图标。右键菜单会根据当前状态显示“标记为独立导出”或“取消独立导出”。导出时自动识别并拆分。



#### 关联图片导出机制



导出 Markdown 时会自动收集并打包备注中引用的本地图片，保证导出包离线可用。流程为：递归遍历所有节点的备注字段；按约定格式匹配图片路径并去重；将服务器相对路径重写为 ZIP 内相对路径（如导出包内 images 目录下保留原文件名）；若无独立导出且无图片则只生成单个 MD，否则生成 ZIP。图片文件缺失时仅跳过并记录告警，不影响 MD 内容导出。ZIP 内包含主 MD、各独立节点 MD 以及 images 目录；主 MD 中对独立节点使用“详见独立文档”的链接引用。后端提供两个工具：一是递归收集节点树中所有备注引用的本地图片路径，二是将内容中的服务器图片路径重写为 ZIP 内相对路径。



### 4.4 分享功能



提供创建分享的接口（请求体含脑图及可选标题，返回是否成功、分享 ID 与访问链接）；通过分享 ID 可打开可编辑的思维导图页面；另提供按分享 ID 更新脑图内容的接口，供分享页编辑后保存使用。



### 4.5 导入功能



- 将 Markdown 内容智能合并到现有思维导图：请求体含内容及可选文件名，由 AI 识别应放置的节点位置，返回是否成功、合并数量及目标节点。

- 批量导入：请求体指定目录，后端导入该目录下所有 MD 文件，返回成功与失败文件列表。

- 上传 XMind 文件：请求为 multipart 表单，包含 xmind 文件及可选标题；后端解析压缩包内的 content 配置文件为脑图并返回结果（当前不支持仅含 content 旧格式的文件）。

- 导入纯文本：请求体含缩进表示层级的文本及可选标题；缩进规则为四个空格或一个 Tab 表示一级；后端转换为思维导图并创建文档，返回文档信息与脑图。



### 4.6 图片上传



提供图片上传接口，请求为 multipart 表单的 file 字段；存储路径按年、月及哈希与扩展名组织；支持 JPEG、PNG、GIF、WebP、BMP，大小限制 5MB；返回是否成功、URL、文件名与大小。



### 4.6 用户认证



- 工号登录：请求体为工号，无密码；成功则返回用户信息与 token，并在 Cookie 中写入 HttpOnly、7 天有效期的 token。

- 管理员密码登录：请求体为密码，当前默认口令建议部署时改为配置项；返回用户信息与 token。

- 游客登录：无请求体，后端生成游客 ID，返回用户信息与 token，其中登录类型为“游客”。

- OAuth 回调：小九 OAuth 回调后重定向到首页并设置 Cookie，token 中登录类型为“小九”。

- 登出：清除登录 Cookie。

- 获取当前用户：返回当前用户信息，未登录则返回 401。



#### OAuth 用户本地映射机制



小九 OAuth 登录时，用 OAuth 返回的工号在本地用户表中查找；若存在则返回本地用户并保留数据库中的角色与权限；若不存在则自动创建本地用户（默认角色为员工），后续可在用户管理中调整角色。字段映射：工号对应 work_id；角色优先使用数据库配置；姓名优先使用数据库中的名称；登录类型支持工号、小九、管理员密码、游客等。

# 项目设计文档 - XMind AI Planner（第 2 部分）



### 4.7 用户管理（仅 admin）



仅管理员可访问。提供获取用户列表、创建用户（请求体含工号、姓名、角色，角色可为管理员、审核员或员工）、更新用户信息（不可修改管理员账号的角色）、删除用户（不可删除管理员账号）等接口。



### 4.8 个人工作区



需登录。提供获取当前用户文档列表、创建新文档（可选标题，默认“新文档”）、按文档 UUID 重命名、按文档 UUID 软删除等接口；重命名与删除要求当前用户为文档所有者。



### 4.9 文档版本历史



提供获取文档版本列表的接口（不含完整内容，节省带宽）；获取指定版本完整内容的接口（用于预览）；以及回滚到指定版本的接口（需文档所有者或管理员，回滚前会先保存当前状态）。



## 5. 前端组件设计



### 5.1 渲染与交互（MindElixir）



画布由 MindElixir 在指定容器上挂载，负责节点布局、连线、拖拽画布、缩放、折叠与展开。数据流为：我们自己的数据经转换为库格式后初始化；用户操作或 AI 结果通过库提供的添加子节点、重塑节点、删除节点等接口更新；导出或保存时从库取数再转回我们的格式。自定义扩展包括：AI 创建整图、AI 展开子节点、AI 重写节点、导出时转格式并提交、右键菜单（自绘，通过库提供的按 ID 查找节点接口获取目标）。



### 5.2 节点备注编辑器（Vditor）



使用 Vditor 作为 Markdown 图文混排编辑器，采用 IR 即时渲染模式，类似所见即所得。通过 CDN 引入其样式与脚本。图片上传：工具栏上传与粘贴上传均通过自定义处理函数调用后端的图片上传接口；服务器按年、月及哈希与扩展名存储，支持 JPEG、PNG、GIF、WebP、BMP，最大 10MB。备注对话框支持标题栏拖拽移动、八方向调整尺寸（最小 400×320）、打开时默认居中且默认尺寸 720×560，标题栏右侧显示当前编辑的节点名称。每次打开备注时创建新的 Vditor 实例，模式为 IR，主题随当前主题切换，语言为中文，工具栏包含标题、加粗、斜体、删除线、引用、列表、链接、上传、表格、代码、撤销重做、编辑模式与全屏等；上传配置指向后端图片上传接口，最大 10MB，接受图片类型，自定义处理函数在收到文件后通过表单提交并插入图片链接。



### 5.3 节点与画布规则



节点尺寸、连线、间距由 MindElixir 默认主题与配置决定；我们仅通过样式覆盖画布背景以与深色主题协调。



### 5.4 主题系统



全站通过 CSS 变量实现主题切换；暗色为默认变量定义在根上，亮色通过指定 data-theme 属性覆盖。本地存储中保存用户偏好（暗色、亮色或跟随系统）；跟随系统时根据系统偏好决定实际主题。变量分类包括：基础色（主色、背景、文字）、表面色（卡片、面板、边框）、语义色（成功、警告、危险）、阴影、半透明毛玻璃、Diff 对比色、徽章等。防闪烁策略：在页面头部内联一段同步脚本，在 DOM 解析前读取本地存储并设置 data-theme，避免首屏白屏或黑屏闪烁；所有主要页面均包含该脚本。主题切换时需同步更新 MindElixir 与 Vditor 的主题（MindElixir 通过 CSS 变量控制背景；Vditor 通过调用其主题设置接口）。核心逻辑包括：初始化主题、获取实际生效主题、获取与保存用户偏好、切换主题、应用主题到 DOM（含过渡动画）、更新 MindElixir 与 Vditor 主题。



## 6. AI 交互设计



### 6.1 生成初始结构 Prompt



提示词要求 AI 扮演专业策划人员，根据给定主题生成思维导图结构：生成四到六个一级分支，每个一级分支下二到四个二级分支；内容简洁，每节点五到二十字；覆盖主题核心方面；返回约定格式的结构数据。



### 6.2 展开子节点 Prompt



提示词要求 AI 根据当前路径、当前节点文本、已有子节点列表及可选额外说明，生成三到五个子节点；与现有子节点不重复、内容符合上下文、每节点五到二十字；返回约定格式。

项目根目录下包含：后端主程序、Python 依赖说明、前端依赖说明（仅用于安装 MindElixir 并将构建产物拷贝到静态目录）、README、需求文档、本设计文档。静态目录下包含前端页面、样式、前端业务与适配脚本，以及本地存放的 MindElixir 构建产物（含主脚本与样式）。另有前端依赖安装目录与 Python 虚拟环境目录。



## 8. 部署说明



### 8.1 环境变量



需配置 AI API 地址、API Key、模型名称等，示例值分别为 OpenAI 的聊天补全地址、以 sk 开头的 Key、以及具体模型名。



### 8.2 启动命令



开发模式：进入项目目录、激活虚拟环境后，以 uvicorn 运行主应用，指定主机与端口并开启热重载。生产模式：通过环境变量传入配置，使用 nohup 在后台运行并将输出重定向到日志文件。



## 9. 数据库设计（v3.0）



### 9.1 存储架构变更



v3.0 将数据存储从 JSON 文件迁移到 SQLite，以提升一致性与完整性、支持用户与工作区及文档版本等能力，并为后续协作打基础。数据库文件位于数据目录下。



### 9.2 数据库表结构



**用户表**：自增主键、工号（唯一）、姓名、角色（默认员工）、创建与更新时间。**工作区表**：自增主键、名称、描述、所有者外键、创建与更新时间。**工作区成员表**：工作区 ID、用户 ID、角色（默认成员）、加入时间，主键为工作区与用户组合。**文档表**：自增主键、UUID（唯一）、标题、所有者、工作区（可为空表示个人文档）、内容（JSON）、软删标记、创建与更新时间。**文档版本表**：自增主键、文档 ID、版本号、内容、修改人、变更摘要、创建时间。**节点备注表**：文档 ID、节点 ID、备注内容、更新人、更新时间，主键为文档与节点组合。



### 9.3 数据迁移



应用启动时自动执行从 JSON 到 SQLite 的迁移：扫描原分享数据目录下的 JSON 文件，将每条数据导入文档表；已存在文档（按 UUID）跳过；迁移结果写入启动日志。



### 9.4 兼容性封装



数据库模块提供文档存储类，保持与原 JSON 存储相同的对外接口：按分享 ID 保存（含标题与脑图）、按分享 ID 获取、更新脑图、删除、列出全部。



### 9.5 数据库模块 API



除上述兼容层外，还提供：初始化数据库与表结构、按工号获取或创建用户、创建文档、按 UUID 获取或更新或删除文档、列出文档、保存与获取文档版本、创建工作区、添加工作区成员等函数。

# 项目设计文档 - XMind AI Planner（第 3 部分）



## 10. 扩展点



节点拖拽排序：监听拖拽开始、经过与放下事件，并更新节点在子节点数组中的顺序。撤销与重做：维护操作历史栈，每次操作前保存快照，撤销时恢复上一快照。多人协作：通过 WebSocket 实时同步，冲突处理可采用 OT 或 CRDT 等方案。



## 11. 认证模块设计



### 11.1 认证流程



用户输入工号后请求登录接口，服务端验证后生成 JWT Token 并设置 Cookie，返回用户信息。后续访问页面时检查 Cookie、解析 JWT 并验证有效性，通过则允许访问，否则跳转登录页。



### 11.2 JWT Token 结构



Token 载荷包含：用户 ID、工号、角色、过期时间（如 7 天）、签发时间。



### 11.3 权限装饰器



提供“需要登录”“需要管理员”“需要审核员或管理员”“可选登录”等装饰器，用于保护接口。



### 11.4 安全考虑



Token 使用 HS256 签名；Cookie 设置 HttpOnly 防 XSS、SameSite 防 CSRF；内网环境不强制 HTTPS。



### 4.10 项目工作区 API



**工作区管理（仅管理员）**：获取工作区列表（管理员可见全部，普通用户仅可见自己参与的）；创建、更新、删除工作区（删除时区内文档移回原作者个人工作区）。**工作区成员管理（仅管理员）**：获取成员列表、添加成员、移除成员。**工作区文档**：获取工作区文档列表、在工作区中创建文档、将文档移动到指定工作区（或移回个人工作区）；权限为管理员或工作区成员，移动需文档所有者或管理员。



## 12. 实时协作架构



### 12.1 WebSocket 连接管理



后端维护“文档 UUID 到（用户 ID 到连接信息）”的映射；提供建立连接（并广播用户列表）、断开连接、向某文档房间广播消息（可排除某用户）、广播在线用户列表等能力。



### 12.2 消息流程



用户 A 发送保存消息到服务端，服务端写入数据库并记录版本历史，再向用户 B 广播更新内容及操作者；用户 A 收到“已保存”及版本号。



### 12.3 前端状态管理



前端维护 WebSocket 实例、重连定时器与重连次数、在线用户列表、防抖保存定时器、上次保存内容（用于变化检测）。核心函数包括：建立与断开连接、处理收到的消息、调度两秒防抖的自动保存、通过 WebSocket 发送保存、根据新内容重新加载文档。



### 12.4 自动保存策略



用户编辑触发保存入口后，先进入两秒防抖；防抖到期后比较当前内容与上次保存内容，若有变化则通过 WebSocket 发送保存请求；服务端保存成功后广播给其他用户。



### 12.5 API 接口



实时协作端点为 WebSocket，路径为“文档 UUID”。认证从 Cookie 取 token，校验 JWT 与文档存在性。消息类型：客户端可发保存（含内容）、光标（节点 ID）、心跳；服务端可发在线用户列表、内容更新（含操作者）、已保存（含版本号）、心跳响应。



## 13. 审核流程设计

### 15.7 阶段 2：对话数据模型设计



#### 15.7.1 数据库表设计



**对话表**：自增主键、对话 UUID（唯一）、文档 ID、用户 ID、标题、状态（默认活跃）、当前上下文节点 ID、创建与更新时间。**消息表**：自增主键、对话 ID、角色（用户、助手、系统）、内容、元数据（JSON，如节点修改指令）、创建时间。**节点修改记录表**：自增主键、对话 ID、消息 ID、节点 ID、修改类型（增、改、删、移）、修改前与修改后值（JSON）、状态（待处理、已接受、已拒绝）、创建时间。



#### 15.7.2 API 设计



**对话管理**：在文档下创建对话（可选标题与上下文节点）；获取文档的对话列表（可按状态筛选）；获取对话详情（含消息与待处理修改）；更新对话（标题、状态、上下文节点）；删除对话（级联删除消息与修改记录）。**消息管理**：发送消息并获取 AI 响应（请求体含内容与可选上下文节点），返回用户消息、助手消息与节点修改列表。**流式消息**：提供流式发送接口，响应为事件流，事件类型包括思考步骤、逐 token、修改列表、完成、错误。**节点修改管理**：更新单条修改状态（接受或拒绝）；批量更新某条消息或全部待处理修改的状态。



#### 15.7.3 前端状态管理



前端新增状态：当前对话、对话列表、待处理节点修改。核心函数：创建对话、加载对话、删除对话、发送 Agent 消息、处理单条或批量修改、将修改应用到脑图。



#### 15.7.4 修改预览 UI



界面展示“建议的修改”数量及“全部应用”“全部撤销”。每条修改为一块卡片：标题为修改类型与节点 ID 或父节点信息；内容区为原内容（删除线）与新内容（高亮）；底部为“Keep”与“Undo”按钮。新增、修改、删除、移动在视觉上用不同图标与背景色区分。

### 15.8 阶段 3：AI 对话集成设计



#### 15.8.1 核心函数



后端提供异步函数：入参为完整脑图、当前上下文节点 ID、用户消息、历史消息列表；返回文本与可选修改列表。



#### 15.8.2 上下文构建流程



先按 ID 在脑图中递归查找上下文节点；再构建从根到该节点的路径（如“项目概述 > 核心功能 > 用户系统”）；然后组装路径、节点内容、备注、子节点列表；最后生成脑图摘要（缩进文本，含节点 ID 与文本，限制深度与每层子节点数）。



#### 15.8.3 AI 系统提示词（System Prompt）



要求 AI 扮演思维导图助手，能够修改节点内容、添加子节点、删除节点、移动节点、优化表述、生成备注。若需修改脑图，须在响应末尾以约定格式输出修改指令：指令为结构化的修改列表，每条含节点 ID、操作类型（更新、添加、删除）及新值（如新文本、新备注）；须放在专用标记的代码块中；节点 ID 必须为实际存在的 ID；添加操作的节点 ID 指父节点；当存在“当前选中节点”时，本条消息中的操作必须使用该 ID，不得使用历史中的其他节点。



#### 15.8.4 AI 调用参数



模型名从环境变量读取；温度为 0.7；最大 token 数 2000；消息列表由系统提示、脑图摘要、上下文节点信息、最近若干条历史与当前用户消息组成。



#### 15.8.5 消息构建顺序



依次为：系统提示词、当前思维导图结构摘要、当前上下文节点及“本条消息操作目标”的节点 ID 强调、最近若干条历史（仅用户与助手）、最新用户消息。



#### 15.8.6 AI 响应解析



调用 AI 接口获响应后，用正则匹配专用代码块，解析出修改列表；从响应文本中移除该块；返回清理后的文本与修改列表（或无）。



#### 15.8.7 脑图摘要生成算法



递归遍历节点，按深度缩进输出“节点 ID 与文本”；限制最大深度与每层子节点数，超出部分以“还有 N 个节点”提示。



### 15.9 阶段 4-5：修改应用与撤销设计



#### 15.9.1 节点修改应用（前端）



根据修改类型调用 MindElixir：更新则改节点文本与备注；添加则在父节点下增加子节点；删除则移除节点；移动暂未实现。应用后触发防抖保存。



#### 15.9.2 修改记录生命周期



AI 返回修改后创建待处理记录；前端展示 Diff 卡片；用户点 Keep 则标记为已接受并应用、点 Undo 则标记为已拒绝；“全部应用”将所有待处理改为已接受并逐个应用；“全部撤销”改为已拒绝。



#### 15.9.3 修改预览 UI 交互



卡片内为修改类型与节点信息、原内容（红色删除线）与新内容（绿色高亮）、Keep 与 Undo 按钮；删除类为被删内容红色高亮。



### 15.10 对话生命周期设计



对话采用懒创建：打开面板不立即创建，用户发第一条消息时才创建并带上当前选中节点 ID。状态流转：新建面板时前端无对话对象；发首条消息时请求创建对话并进入活跃；可关闭、恢复或删除对话；关闭即更新状态，恢复即加载消息与修改，删除即级联删除。历史管理：可查看文档下对话列表（标题、消息数、时间）、恢复某条对话、新建（前端清空）、删除。



### 15.11 前端状态管理（Cursor 风格版）

### 15.12 前端核心函数目录（Cursor 风格版）

### 15.13 消息渲染设计（Cursor 风格版）



消息类型与样式：用户为浅色背景与左蓝边框、全宽块；AI 为无背景纯文本、全宽流；系统为灰色小字；错误为红底；步骤为旋转图标加文字、行内；Diff 为边框卡片、全宽可折叠。消息结构为无头像与气泡：用户与 AI 消息均为块级容器内嵌内容区；AI 消息可带消息 ID 属性；内联 Diff 紧跟 AI 消息，含标题行（类型与节点信息、悬停显示的操作按钮）与内容区（删除行与新增行）。Markdown 格式化规则：双星号包络为加粗，反引号包络为行内代码，短横线列表转为圆点，换行转为换行标签。



### 15.14 SSE 流式响应架构



流式端点为“对话 UUID”下的“消息流”路径，请求为 JSON，响应为事件流类型。事件协议：思考步骤（含步骤文案）、逐 token（含当前 token）、修改列表（解析出的修改）、完成（含消息 ID 与节点修改）、错误（含错误信息）。后端流程：校验对话与权限、保存用户消息、先发送“正在分析脑图”等步骤、构建上下文、再发送“正在生成回复”、以流式调用 AI、逐行读取并转发 token、累积全文后解析修改块、保存 AI 消息与修改记录、发送修改事件与完成事件。前端流程：请求流式端点、创建 AI 消息占位与流式光标、读取流、解析事件：步骤对应步骤指示器、token 追加并格式化、修改暂存、完成时移除光标并渲染内联 Diff 与更新操作栏、错误时展示错误；若流式不可用则回退到非流式发送。原有非流式消息接口保留，前端优先用流式，失败则回退。



## 16. 自动化开发闭环设计（v3.2）



### 16.1 系统架构



脑图编辑器中：节点编辑、自动保存与“自动开发”开关共同驱动节点变更事件，进入任务队列。任务队列按状态机流转：等待、编码中、差异就绪、同步完成、构建完成、完成（以及需确认、失败、取消、回滚等）。任务执行器包含：需求分析、AI 编码、Diff 生成、同步到测试空间等步骤；并依赖 Artifacts 存储（对话记录、单文件 diff、统一 patch、清单文件）与临时提交工作区（按任务 ID 落盘、支持合并包与清理）。



### 16.2 数据库表设计



**开发任务表**：任务 ID（主键）、工作区 ID、脑图文档 ID、状态、触发类型与触发节点 ID、需求描述、分析结果（JSON）、编码结果（JSON）、Diff 摘要、同步结果、构建结果、错误信息、需确认原因、创建与更新及开始与结束时间。**工作区扩展字段**：是否启用自动开发、测试空间路径、绑定导入根（JSON 数组）、导出根路径。



### 16.3 状态机设计



从“等待”开始执行后进入“编码中”；若需人工确认则进入“需确认”，用户确认后回到“编码中”；编码成功进入“差异就绪”，失败进入“失败”（可重跑回“等待”）。从“差异就绪”同步后进入“同步完成”，再经构建验证进入“构建完成”，最后进入“完成”；“完成”可回滚到“已回滚”。



### 16.4 核心模块设计



**任务执行器**：维护任务状态常量；执行单任务时按当前状态推进：等待时做需求分析，若需确认则更新为需确认并返回，否则进入编码；编码阶段调用 AI 编码，失败则标记失败，成功则进入差异就绪；差异就绪后执行同步并进入同步完成；同步完成后做构建验证并进入构建完成；构建完成后标记完成并提交到工作区。**Artifacts 模块**：初始化任务产物目录（含 diff 子目录）；追加对话记录到按行存储的文件；按文件保存 diff（旧新内容生成统一 diff 格式）；合并所有 diff 生成统一 patch；更新清单文件。**提交工作区模块**：提交根目录固定；提交任务时将产物目录复制到以任务 ID 命名的子目录并写入任务元数据；支持按多任务创建合并包 ZIP；支持按天数清理旧提交。



### 16.5 API 端点设计



提供：文档自动开发前置校验（是否在工作区且配置完整）；工作区任务列表与手动创建任务；任务详情、取消、重跑、确认需确认、回滚；任务产物摘要、对话记录、diff 列表与详情、patch 下载、清单、手动提交；已提交任务列表与详情、创建合并包、清理旧任务。



### 16.6 need_confirm 触发规则



当分析结果中“待删除文件数”超过 3、或“待修改文件数”超过 10、或“置信度”低于 0.7、或存在“风险操作”标记时，返回需要人工确认及原因文案；否则返回不需要确认。



### 16.7 Artifacts 目录结构



每个任务 ID 下：对话记录为按行 JSON 文件；清单为 JSON；统一 patch 为单文件；diff 子目录下为索引与若干单文件 diff 文件。



### 16.8 manifest 结构



清单包含任务 ID、创建与结束时间、耗时、状态、各阶段（分析、编码、diff、同步、构建）的开始与结束及状态、变更文件列表（路径、操作类型、增删行数）、已同步文件列表、汇总统计（总增删行数、对话轮数等）。

## 17. 文件目录面板 + 脑图关联（v3.3）



### 17.1 架构设计



编辑器左侧为文件目录面板，中间为 MindElixir 脑图画布，右侧为 Agent 对话面板。目录面板可展示多个绑定导入根；节点可关联文件或子脑图；画布与后端保存同步，Agent 与 AI 通过 SSE 通信。目录面板宽度可拖拽调整。



### 17.2 节点数据模型扩展



在原有节点字段基础上增加两个可选字段：**文件引用** 与 **关联文档 UUID**。文件引用包含：相对路径、绝对导入根、类型（文件或目录）、扩展名。关联文档 UUID 表示当前节点关联的另一份脑图文档。前端用三个全局映射分别维护：节点 ID 到文件引用、节点 ID 到关联文档 UUID、节点 ID 到备注（原有）。



### 17.3 文件目录面板实现



打开文档时若工作区配置了绑定导入根，则展示文件目录面板。目录树通过“工作区 ID + 根路径 + 子路径”的接口获取；后端在返回前按全局排除名单过滤（如默认排除某版本控制目录），规则支持精确名与通配符。前端对工作区导入根字段做兼容（多种命名形式统一为同一字段）。交互：点击文件夹展开或折叠（子目录懒加载）；右键文件或文件夹可选择“添加为节点”；支持将文件从面板拖拽到脑图节点上作为子节点添加；面板可收起或展开，可拖拽右边缘调宽。



### 17.4 MD 文件编辑器



复用与备注相同的编辑器（Vditor IR 模式），仅数据来源与保存方式不同：备注从节点备注映射读取、通过脑图保存接口写入；文件从“工作区文件预览”接口读取、通过“工作区文件写入”接口保存；弹窗标题分别为“编辑备注”与“编辑文件”。文件编辑弹窗与备注弹窗一致支持拖拽移动与八方向调整尺寸；编辑器高度随弹窗自适应。



### 17.5 脑图关联操作流程



**绑定现有脑图**：节点右键选择“绑定现有脑图”，弹出选择器列出当前工作区文档列表，选择后将该文档 UUID 写入节点关联并保存脑图。**导出为新脑图**：节点右键选择“导出为新脑图”，后端将该节点子树深拷贝为新文档（同工作区或私人空间），并自动将原节点的关联文档设为新文档 UUID，前端刷新状态。**收回关联脑图**：节点右键选择“收回关联脑图”，后端读取关联脑图的子节点并合并到当前节点，然后清除该节点的关联文档 UUID（不删除原脑图），前端重新加载文档。



### 17.6 新增 API



- 获取工作区文档列表：返回简洁列表（UUID、标题、所有者 ID）。

- 将节点子树导出为新文档：请求体含节点 ID 及是否清空当前子节点；后端深拷贝子树为新文档并设置原节点关联。

- 收回关联脑图：请求体含节点 ID；后端合并关联脑图子节点到该节点并清除关联。

- 分享接口扩展：在返回中增加工作区信息（ID、名称、绑定导入根、导出根路径）。
